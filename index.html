Crear repositorio público

GIT

Primero, descargar GIT (Windows o Mac). En linux no, porque ya lo tiene integrado.

    Seleccionar la opción:
  • Use git from the Windows Command Prompt le agrega la capacidad de trabajar con GIT dentro de la consola de Windows.
  • Select components: Git Bash Here

Configurar GIT

  1. Abrir Git Bash, sin importar la ubicación
  2. Establecer un nombre de usuario: git config --global user.name "Gabriela Bustos"
  3. Configurar E-mail: git config --global user.email "viajacontumente@gmail.com"
  4. Crear clave SSH:
    • Ingresar ls -al ~/.ssh para ver si hay claves SSH presentes. También pueden encontrarse en: C:/Usuarios/(nombre de usuario)/.ssh
    • Generar clave (utilizando el e-mail que pusimos en Github): ssh-keygen -t rsa -b 4096 -C "viajacontumente@gmail.com" entonces me va a pedir una passphrase: no ponemos nada (si llegamos a escribir contraseña, en git no se muestra mientras escribimos). Enter, enter, enter.
  5. Vamos a la carpeta .ssh y abrimos con Visual Studio code el archivo .pub
  6. Copiamos la clave
  7. En Github, en Settings vamos a SSH and GPG keys: New SSH key ahí pegamos el código ssh y podemos poner un título descriptivo para saber cuál/dónde/cuándo creamos esta clave.
  8. Poner contraseña de Github

Podemos luego clonar repositorios utilizando ssh.

GITHUB

Github Pages nos permite tener sitios web estáticos que están basados en un repositorio.
Github nos va a permitir crear un repositorio que va a estar relacionado con nuestra página web. Y todo lo que pongamos en ese repositorio, lo vamos a poder publicar en internet.
Github Pages no reemplaza a los hostings compartidos, porque cuando contratamos un hosting, estamos pagando por un espacio en un servidor, y eso me va a dar acceso a otras características como por ej poder ejecutar un lenguaje de back-end, trabajar con archivos y bases de datos, etc. No podríamos instalar Wordpress o Woocommerce.

Conectar a Github

Va a pedir que pongamos usuario y contraseña.

After you run that command, you should get feedback that an empty Git repository was initialized for your project.
To initialize a new local Git repository we need to write on the cmd: git init
If anything happens, everything is okay!

Write git add . to track all your files.

And then add: git commit -m "Initial commit"
This is really important step.
When that’s done, it means that we successfully prepared our new local repository to be pushed to GitHub!

If you need more space in cmd, just write: clear. And it will be done.

For the Repository name, we can specify the same project name as the local repository that we are using before:
git remote add origin https://github.com/gabib89/tutorial-github.git
Those was all the steps.
